# 4주차 정리

## API

> 라이브러리에 접근하기 위한 규칙들을 정의

- Interface

> 서로 다른 두 시스템, 장치, 소프트웨어 간의 상호 작용을 허용하도록 하는 중재나 매개체가 되어주는 역활

- REST API, RESTful API

> HTTP 형식을 지킨 API - REST API
> HTTP 형식을 매우 매우 잘 지킨 API - RESTful API

### URL

- REST API URL 규칙

  1. 대문자는 쓰지 않음

  2. 언더바 X, 하이픈(-) O

  3. 마지막 / 는 포함하지 않음

  4. **`행위`를 포함하지 않음**

  5. 파일 확장자 X

  6. 복수형을 쓴다. (통일감, 어감)
